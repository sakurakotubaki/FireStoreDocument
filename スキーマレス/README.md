# スキーマレスについて
スキーマレスとは、データベースにおいて、データの構造をあらかじめ定義しないことを指す。

スキーマレスなデータベースは、データの構造をあらかじめ定義しないため、データの構造を変更することが容易である。

コレクションは、RDBのテーブルと違ってスキーマ情報は一切持っていません。
そのため、コレクションには、様々な構造のドキュメントを格納することができます。

スキーマの情報があるRDBは、データの構造を変更することが難しいです。なぜなら、列の名前や型を変更すると、既存のデータに影響が出るからです。

RDBの構造はテーブルで表現するとこんな感じです。エクセルの表のようなイメージです。

| id | name | age |
| --- | --- | --- |
| 1 | John | 30 |
| 2 | Mary | 25 |

スキーマがないFirestoreだとこんな感じになります。

```json
{
  "id": 1,
  "name": "John",
  "age": 30
}
```

```json
{
  "id": 2,
  "name": "Mary",
  "age": 25
}
```
